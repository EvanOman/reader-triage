# Zero-Inflated and Count Models for Article Engagement Prediction

## Executive Summary

Predicting article engagement measured by highlight counts is fundamentally a zero-inflated count data problem: most articles receive zero highlights (either unread or read without highlighting), while a smaller subset receives 1-20+ highlights. Standard regression models fail here because they assume continuous, normally distributed outcomes. This report surveys the full landscape of statistical and machine learning approaches for this problem -- from classical Poisson and Negative Binomial regressions through Zero-Inflated Poisson (ZIP), Zero-Inflated Negative Binomial (ZINB), Hurdle models, and Tobit models, to modern deep learning architectures that parameterize zero-inflated distributions. For our specific setting (~hundreds of articles, text embedding features available, a well-understood two-stage data-generating process), the Hurdle model emerges as the most interpretable and robust choice, while ZINB provides a useful comparison. We also cover strategies for transforming predicted counts into 0-100 quality scores and practical model selection diagnostics.

## 1. Statistical Count Models: A Progressive Framework

### 1.1 Poisson Regression (Baseline)

The Poisson model is the simplest count regression. It assumes that the response variable Y follows a Poisson distribution where the log of the expected count is a linear function of the predictors:

```
log(E[Y|X]) = X*beta
```

The critical assumption is **equidispersion**: Var(Y) = E[Y]. In practice, count data almost always violates this -- the variance exceeds the mean (overdispersion). For highlight counts, where many articles have 0 highlights and a few have 10+, the variance will vastly exceed the mean. The Poisson model serves only as a baseline to demonstrate why more complex models are needed.

**Diagnostics**: Compute the Pearson chi-squared dispersion statistic (Pearson chi2 / df_resid). Values substantially greater than 1.0 indicate overdispersion and the need to move beyond Poisson ([UCLA OARC, 2024](https://stats.oarc.ucla.edu/r/dae/zip/)).

### 1.2 Negative Binomial Regression

The Negative Binomial (NB) model relaxes the equidispersion constraint by adding a dispersion parameter alpha:

```
Var(Y) = E[Y] + alpha * E[Y]^2     (NB2 parameterization)
```

When alpha = 0, this reduces to Poisson. The NB model is the workhorse of count data analysis and handles overdispersion well. [Allison (2012)](https://statisticalhorizons.com/zero-inflated-models/) argues that in many cases, a standard Negative Binomial model is sufficient even with many zeros -- the NB distribution naturally accommodates higher zero probabilities than Poisson through its overdispersion parameter.

Paul Allison's influential argument is worth quoting: "There's nothing in the data that will tell you whether some zeros are structural, and some are sampling. That has to be decided on theoretical grounds." His recommendation is to start with NB and only move to zero-inflated models when you have substantive theory supporting two distinct zero-generating processes ([Statistical Horizons, 2012](https://statisticalhorizons.com/zero-inflated-models/)).

The **Cameron-Trivedi test** formally tests whether overdispersion is present. It regresses an auxiliary variable on the squared fitted values from the Poisson model. A significant positive coefficient confirms overdispersion and justifies using NB over Poisson ([Cameron & Trivedi, 1990](https://www.sciencedirect.com/science/article/abs/pii/030440769090013K)).

### 1.3 Zero-Inflated Poisson (ZIP)

The ZIP model assumes the data is generated by a mixture of two processes:

1. A **point mass at zero** with probability pi (the "always-zero" or structural-zero process)
2. A **Poisson distribution** with probability (1 - pi)

The probability mass function is:

```
P(Y = 0) = pi + (1 - pi) * exp(-mu)
P(Y = y) = (1 - pi) * exp(-mu) * mu^y / y!     for y > 0
```

The inflation parameter pi is typically modeled via a logistic regression on its own set of covariates (the "inflation equation"), which may differ from the covariates in the count equation. This is the model proposed by [Lambert (1992)](https://www.jstor.org/stable/1269547) for manufacturing defect data.

ZIP is appropriate when you believe the data contains excess zeros beyond what a Poisson distribution would predict, but the non-zero counts are well-described by a Poisson. In practice, if the data is also overdispersed (which highlight counts almost certainly are), ZIP will still underfit. This motivates ZINB.

### 1.4 Zero-Inflated Negative Binomial (ZINB)

ZINB combines the mixture approach of ZIP with the flexibility of the Negative Binomial:

```
P(Y = 0) = pi + (1 - pi) * NB(0; mu, alpha)
P(Y = y) = (1 - pi) * NB(y; mu, alpha)     for y > 0
```

This model simultaneously handles both excess zeros (via the inflation component) and overdispersion (via the NB dispersion parameter). Multiple comparative studies have found ZINB to be the best-performing model for data exhibiting both characteristics. [Alshkaki (2025)](https://dialoguessr.com/index.php/2/article/view/887) reports that ZINB consistently achieves the lowest AIC, BIC, RMSE, and MAE across a range of count data scenarios. [Gul et al. (2023)](https://link.springer.com/article/10.1186/s13063-023-07648-8) similarly found ZINB outperformed standard NB in a clinical trial analysis.

The inflation equation in ZINB models is particularly powerful when you have a known predictor of structural zeros. In our case, `reading_progress` is such a predictor -- articles with zero reading progress are very likely to be structural zeros (the user never opened them).

### 1.5 Hurdle Models

The Hurdle model (also called the two-part model) takes a fundamentally different approach from zero-inflation. Instead of mixing two distributions, it separates the modeling into two completely independent components:

1. **Part 1 (Binary model)**: A logistic regression modeling P(Y > 0), the probability of "crossing the hurdle"
2. **Part 2 (Truncated count model)**: A zero-truncated Poisson or Negative Binomial modeling E[Y | Y > 0], the expected count given positive engagement

The probability mass function is:

```
P(Y = 0) = f_1(0)                     (from the binary model)
P(Y = y) = [1 - f_1(0)] * f_t(y)      for y > 0
```

where f_t is the truncated count distribution (renormalized to sum to 1 over positive integers).

This was originally proposed by [Mullahy (1986)](https://www.sciencedirect.com/science/article/abs/pii/0304407686900021). The key conceptual advantage is that **all zeros are treated as coming from a single process**. There is no latent mixture -- the binary model cleanly separates "did the user engage?" from "how much did they engage?"

[Neelon (2021)](https://link.springer.com/article/10.1186/s40488-021-00121-4) provides a thorough simulation study comparing hurdle and zero-inflated models. Key findings:

- Hurdle models can handle **zero-deflation** (fewer zeros than expected at some covariate levels), whereas ZI models cannot -- they produce parameter estimates of infinity when zero-deflation occurs
- When the true data-generating process involves zero-inflation, hurdle models still fit well with binary covariates but may lose power with continuous covariates where structural and sampling zeros differ substantially
- When the goal is **prediction** rather than understanding the zero-generating mechanism, both model families perform comparably

### 1.6 Model Comparison Summary

| Model | Zeros | Overdispersion | Parameters | When to Use |
|-------|-------|---------------|------------|-------------|
| Poisson | Underestimates | Assumes none | k+1 | Baseline only |
| Negative Binomial | Moderate fit | Handles well | k+2 | First real model; often sufficient |
| ZIP | Explicit modeling | No | 2k+2 | Excess zeros, no overdispersion |
| ZINB | Explicit modeling | Yes | 2k+3 | Excess zeros + overdispersion |
| Hurdle (NB) | Two-part modeling | Yes (in count part) | 2k+3 | Zeros from known single process |

## 2. Tobit Models and Corner Solutions

The Tobit model ([Tobin, 1958](https://en.wikipedia.org/wiki/Tobit_model)) was designed for **censored continuous data** -- situations where a latent continuous variable Y* is observed as Y = max(0, Y*). The classic example is household expenditure on durable goods: the true "desire to spend" is continuous, but we observe zero when the desire is negative.

```
Y* = X*beta + epsilon,    epsilon ~ N(0, sigma^2)
Y = max(0, Y*)
```

**Why Tobit is generally not appropriate for count data**: Tobit assumes an underlying continuous latent variable that is censored at zero. Highlight counts are inherently discrete -- you cannot have 0.7 highlights. As [Amore (2021)](https://sms.onlinelibrary.wiley.com/doi/full/10.1002/gsj.1363) notes, applying Tobit to discrete dependent variables can produce biased and inconsistent estimates. The model also assumes normally distributed errors, which is violated by the skewed, discrete nature of count data.

**When Tobit might be relevant**: If you transform counts to a continuous engagement score (e.g., highlight density as a continuous measure), a Tobit-like model could apply. However, even then, hurdle models or two-part models are generally preferred because they do not assume the same coefficients drive the zero/non-zero decision and the magnitude of positive values ([Cameron & Trivedi, Chapter 16](https://cameron.econ.ucdavis.edu/mmabook/transparencies/ct16_tobit.pdf)).

The **Cragg model** (sometimes called the double-hurdle model) generalizes Tobit by allowing different parameters in the participation equation and the amount equation -- making it essentially a hurdle model for continuous outcomes. If you need a continuous-outcome analog of the hurdle model, the Cragg model or the standard two-part model are better choices than Tobit ([Greene, Chapter 19](https://pages.stern.nyu.edu/~wgreene/DiscreteChoice/Readings/Greene-Chapter-19.pdf)).

## 3. Combining Count Models with Embedding Features

Our codebase already uses `nomic-ai/nomic-embed-text-v1.5` for generating 768-dimensional article embeddings. The question is how to incorporate these high-dimensional features into count regression models.

### 3.1 Dimensionality Reduction + GLM

The most practical approach for small datasets:

1. **Generate embeddings** for each article (title + first N words of content)
2. **Reduce dimensionality** via PCA, UMAP, or an autoencoder to 5-20 components
3. **Use reduced components as features** in the count model (NB, ZINB, or Hurdle)

This approach has been validated by [Rodriguez et al. (2023)](https://www.cambridge.org/core/journals/american-political-science-review/article/embedding-regression-models-for-contextspecific-description-and-inference/4C90013E5C714C8483ED95CC699022FB), who propose "Embedding Regression" -- feeding embedding-derived features into regression models while maintaining valid statistical inference. The key insight is that the embeddings capture semantic content that structured features (word count, author) miss.

```python
from sklearn.decomposition import PCA
import statsmodels.api as sm

# Reduce 768-dim embeddings to 10 components
pca = PCA(n_components=10, random_state=42)
embedding_components = pca.fit_transform(article_embeddings)

# Combine with structured features
X = np.column_stack([
    info_score,           # AI-generated score
    log_word_count,       # Article length
    embedding_components  # 10 PCA components from embeddings
])

X_with_const = sm.add_constant(X)
model = sm.NegativeBinomial(y_highlights, X_with_const).fit()
```

### 3.2 Embedding Features in the Hurdle Model

A particularly elegant approach uses different embedding features for each part of the hurdle:

- **Binary part**: Embeddings predict whether an article is the "type" that gets highlighted at all (captures topical relevance, writing style)
- **Count part**: Embeddings predict intensity among engaged articles (captures richness, quotability)

This aligns with research by [Tian et al. (2024)](https://arxiv.org/html/2411.14708v1) showing that LLM embeddings can serve as effective regression features, sometimes outperforming traditional feature engineering for high-dimensional prediction tasks.

### 3.3 Regularization for Small Samples

With ~hundreds of articles and 768-dimensional embeddings, overfitting is a primary concern. Recommended strategies:

- **L1/L2 regularization**: Use `statsmodels` penalized models or `sklearn` pipelines
- **Feature selection**: Keep only PCA components explaining >1% variance each
- **Cross-validation**: Use leave-one-out or 5-fold CV for model comparison given the small N
- **Bayesian priors**: Place informative priors on embedding coefficients to prevent extreme estimates

### 3.4 End-to-End Embedding Regression

For a fully neural approach (discussed further in Section 4), feed raw embeddings into a neural network that outputs the parameters of a ZINB or Hurdle distribution. This avoids the information loss of PCA but requires more data to train effectively.

## 4. Deep Learning Approaches to Zero-Inflated Count Prediction

### 4.1 Neural Network Count Models

The fundamental idea is straightforward: replace the linear predictor (X*beta) in a GLM with a neural network, while keeping the distributional assumption (Poisson, NB, ZINB) as the output layer's loss function.

[Haghani & Sedehi (2020)](https://pmc.ncbi.nlm.nih.gov/articles/PMC7189957/) demonstrated this with a multilayer perceptron applied to zero-inflated blood donation count data. Their ANN achieved MSE of 0.27 compared to 2.53 for ZINB, 9.89 for NB, and 4.05 for Poisson regression on test data. The architecture was a single hidden layer with 11-18 nodes trained with backpropagation.

The advantage of neural networks is that they make no parametric assumptions about the relationship between features and the count distribution parameters. The disadvantage is they require more data and provide less interpretable coefficients.

### 4.2 Deep Generative Models (scVI Architecture)

The most sophisticated deep learning approach to zero-inflated count data comes from single-cell genomics. [Lopez et al. (2018)](https://pmc.ncbi.nlm.nih.gov/articles/PMC6289068/) introduced **scVI** (single-cell Variational Inference), which uses a variational autoencoder (VAE) with a ZINB observation model:

1. An **encoder network** maps high-dimensional input to a low-dimensional latent space z ~ N(0, I)
2. A **decoder network** maps z to the parameters of a ZINB distribution: mean mu, dispersion theta, and zero-inflation probability pi
3. Training maximizes the evidence lower bound (ELBO) via stochastic gradient descent

The ZINB loss function in PyTorch is:

```python
def zinb_loss(x, mu, theta, pi):
    """Negative log-likelihood of ZINB distribution."""
    nb_log_prob = (
        torch.lgamma(x + theta) - torch.lgamma(theta) - torch.lgamma(x + 1)
        + theta * torch.log(theta / (theta + mu))
        + x * torch.log(mu / (theta + mu))
    )

    # Zero-inflation component
    zero_prob = pi + (1 - pi) * torch.exp(theta * torch.log(theta / (theta + mu)))
    nonzero_prob = (1 - pi) * torch.exp(nb_log_prob)

    loss = -torch.where(
        x < 1e-8,
        torch.log(zero_prob + 1e-8),
        torch.log(nonzero_prob + 1e-8)
    )
    return loss.mean()
```

Interestingly, the scVI team later found that a plain Negative Binomial (without zero-inflation) often performs comparably, as the NB component alone can explain most zeros through low mean values ([scVI blog, "Should we zero-inflate scVI?"](https://scvi-tools.org/blog/zero-inflation/)). This echoes Allison's argument for traditional count models.

### 4.3 Gradient Boosted Trees with ZINB Loss

[Hu et al. (2023)](https://arxiv.org/abs/2307.07771) propose Zero-Inflated Poisson Boosted Trees, integrating ZIP/ZINB loss functions into gradient boosting frameworks (XGBoost, LightGBM, CatBoost). The approach outperforms standard regression tree methods for insurance claim frequency data, which shares the zero-inflated structure of our problem.

The hybrid approach is attractive: tree-based models naturally handle non-linear feature interactions and require less tuning than neural networks, while the distributional loss function captures the zero-inflated nature of the outcome.

### 4.4 Two-Stage ML Pipeline

A practical machine learning alternative to formal zero-inflated models:

```python
from sklearn.ensemble import GradientBoostingClassifier, GradientBoostingRegressor

# Stage 1: Binary classifier -- will this article get any highlights?
clf = GradientBoostingClassifier()
clf.fit(X_train, (y_train > 0).astype(int))

# Stage 2: Regression on positive-only data -- how many highlights?
pos_mask = y_train > 0
reg = GradientBoostingRegressor()
reg.fit(X_train[pos_mask], y_train[pos_mask])

# Combined prediction
p_positive = clf.predict_proba(X_test)[:, 1]
count_if_positive = reg.predict(X_test)
expected_count = p_positive * count_if_positive
```

This is the ML analog of the hurdle model. [Putatunda & Rama (2019)](https://www.researchgate.net/publication/24083648_Decision_tree_approaches_for_zero-inflated_count_data) discuss decision tree approaches for zero-inflated count data, and the two-stage pipeline approach has been widely adopted in industry for similar problems.

[Barmoudeh & Mukherjee (2023)](https://link.springer.com/article/10.1007/s40745-023-00464-6) compared ML techniques (Random Forest, SVM, KNN, ANN) against count regression models (Poisson, NB, ZIP, ZINB) on zero-inflated ecological count data and found ML models generally performed better in terms of predictive accuracy, though at the cost of interpretability.

### 4.5 Practical Considerations for Small Datasets

With only hundreds of observations, deep learning approaches face significant challenges:

| Approach | Min Observations | Overfitting Risk | Interpretability |
|----------|-----------------|------------------|-----------------|
| Poisson/NB GLM | 30-50 | Low | High |
| ZIP/ZINB | 150-300 | Moderate | Moderate |
| Hurdle model | 100-200 | Moderate | High |
| Random Forest two-stage | 200+ | Moderate (with CV) | Low |
| XGBoost + ZINB loss | 500+ | High without tuning | Low |
| Neural net + ZINB loss | 1000+ | Very high | Very low |
| VAE (scVI-style) | 5000+ | Extreme for small N | Very low |

**Recommendation**: For ~hundreds of articles, deep learning is premature. Stick with GLMs or the two-stage ML pipeline with gradient boosting. If the dataset grows to thousands, reconsider neural approaches.

## 5. Two-Part Models: The Logistic Gate + Count Model Architecture

### 5.1 Conceptual Motivation

The two-part model is the practical implementation of the hurdle concept. It is particularly well-suited to our problem because the zeros have a clear, interpretable meaning:

- **Zero highlights**: Either (a) the article was never opened, (b) the article was opened but not read substantively, or (c) the article was read but nothing was worth highlighting
- **Positive highlights**: The article was read and contained material worth saving

This maps naturally to a two-part decision:
1. **Gate**: Is this article worth engaging with? (binary)
2. **Intensity**: Given engagement, how rich is the content? (count)

### 5.2 Implementation with Separate Models

The hurdle/two-part model decomposes naturally into two independent estimation problems, which simplifies both implementation and interpretation:

```python
from sklearn.linear_model import LogisticRegression
import statsmodels.api as sm
import numpy as np

class HurdleModel:
    """Two-part model: logistic gate + truncated negative binomial."""

    def __init__(self):
        self.gate = LogisticRegression(penalty='l2', C=1.0)
        self.count_model = None

    def fit(self, X, y):
        # Part 1: Binary gate
        y_binary = (y > 0).astype(int)
        self.gate.fit(X, y_binary)

        # Part 2: Truncated count model (fit on positives only)
        pos_mask = y > 0
        if pos_mask.sum() >= 10:
            X_pos = sm.add_constant(X[pos_mask])
            y_pos = y[pos_mask]
            self.count_model = sm.NegativeBinomial(
                y_pos, X_pos, loglike_method='nb2'
            ).fit(maxiter=100, disp=False)
        return self

    def predict(self, X):
        p_positive = self.gate.predict_proba(X)[:, 1]
        if self.count_model is not None:
            X_with_const = sm.add_constant(X)
            count_pred = self.count_model.predict(X_with_const)
        else:
            count_pred = np.ones(len(X))
        return p_positive * count_pred
```

### 5.3 Using Different Features for Each Part

A major advantage of the two-part model is that different predictors can drive each component:

**Gate model** (will the user engage?):
- `reading_progress` -- strongest predictor of structural zeros
- `category` -- some content types (emails, newsletters) rarely get highlighted
- `word_count` -- very short articles may not warrant highlighting
- Topic embeddings -- captures whether the topic is in the user's interest area

**Count model** (given engagement, how many highlights?):
- `info_score` -- overall quality assessment
- `specificity_score` -- quotable passages
- `actionability_score` -- applicable frameworks
- Content embeddings -- captures richness and depth
- `word_count` (as offset) -- longer articles have more opportunity for highlights

### 5.4 Interpreting Two-Part Model Coefficients

Each part yields its own set of coefficients with distinct interpretations:

- **Gate coefficients** (log-odds): A coefficient of 0.03 for `info_score` means each 1-point score increase is associated with exp(0.03) = 1.03x odds of any engagement
- **Count coefficients** (log-rate): A coefficient of 0.02 for `info_score` means each 1-point score increase is associated with exp(0.02) = 1.02x more highlights among engaged articles

The **combined marginal effect** is the product: the expected count change from a 1-unit predictor increase depends on both the change in engagement probability and the change in count intensity.

## 6. Transforming Counts to Continuous Scores (0-100 Mapping)

Our system produces scores on a 0-100 scale, but the ground truth (highlight counts) is a zero-inflated integer. Several strategies exist for mapping between these scales.

### 6.1 Quantile-Based Mapping

The most robust approach uses the empirical cumulative distribution function (CDF) of highlight counts to map to a 0-100 percentile scale:

```python
from sklearn.preprocessing import QuantileTransformer

# Fit on observed highlight distribution
qt = QuantileTransformer(
    n_quantiles=min(len(y), 100),
    output_distribution='uniform',
    random_state=42
)
scores_0_1 = qt.fit_transform(y.reshape(-1, 1))
scores_0_100 = scores_0_1 * 100
```

This approach ([scikit-learn QuantileTransformer docs](https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.QuantileTransformer.html)) is robust to outliers because it operates on ranks. However, with heavily zero-inflated data, all zero-count articles map to the same low score, which may not differentiate between "never opened" and "read but not highlighted."

### 6.2 Expected Count to Score via Sigmoid Mapping

Map the model's predicted expected count through a sigmoid function calibrated to your desired score distribution:

```python
import numpy as np

def count_to_score(predicted_count, midpoint=3.0, steepness=1.0):
    """Map expected highlight count to 0-100 score via logistic function.

    Args:
        predicted_count: Expected highlight count from the model
        midpoint: Count value that maps to score 50
        steepness: Controls how quickly scores transition from low to high
    """
    score = 100 / (1 + np.exp(-steepness * (predicted_count - midpoint)))
    return score
```

The `midpoint` and `steepness` parameters are calibrated from the observed distribution. If the median positive highlight count is 3, setting midpoint=3 means a predicted count of 3 maps to score 50.

### 6.3 Two-Part Score Composition

The most interpretable approach for our system mirrors the two-part model structure:

1. **Engagement probability** (0-1): From the gate model, P(any highlights)
2. **Conditional intensity** (0-1): From the count model, normalized E[Y|Y>0]
3. **Combined score**: Weighted combination

```python
def two_part_score(p_engage, expected_count_if_positive, max_expected=15):
    """Combine gate probability and count intensity into 0-100 score.

    The gate gets more weight because the binary engagement decision
    is the primary signal in zero-inflated data.
    """
    intensity = min(expected_count_if_positive / max_expected, 1.0)

    # 60% weight on engagement probability, 40% on intensity
    raw_score = 0.6 * p_engage + 0.4 * intensity
    return raw_score * 100
```

### 6.4 Log-Linear Mapping with Floor/Ceiling

For a simpler approach that handles the heavy right tail:

```python
def log_count_to_score(count, zero_score=10, max_score=95):
    """Map count to score using log transformation.

    0 highlights -> zero_score
    1 highlight  -> ~30
    3 highlights -> ~50
    10 highlights -> ~75
    20+ highlights -> ~90+
    """
    if count == 0:
        return zero_score
    log_count = np.log1p(count)
    log_max = np.log1p(20)  # Soft ceiling
    normalized = min(log_count / log_max, 1.0)
    return zero_score + normalized * (max_score - zero_score)
```

### 6.5 Calibration via Isotonic Regression

If you have a historical dataset of (predicted_score, actual_highlights) pairs, use isotonic regression to learn a monotonic mapping that calibrates predicted scores to match observed engagement rates:

```python
from sklearn.isotonic import IsotonicRegression

# Fit calibration function
iso = IsotonicRegression(out_of_bounds='clip')
iso.fit(predicted_scores, actual_highlights)

# Apply calibration
calibrated_scores = iso.predict(new_predicted_scores)
```

This is particularly useful for post-hoc calibration of the existing AI scoring system.

## 7. Model Selection: How to Choose Between ZIP, ZINB, and Hurdle

### 7.1 Decision Framework

The choice between models should be driven by both **theory** and **statistical diagnostics**:

```
Step 1: Is there overdispersion? (Var >> Mean)
  |
  Yes --> Prefer NB-based models (NB, ZINB, Hurdle-NB)
  No  --> Poisson-based may suffice (Poisson, ZIP, Hurdle-P)

Step 2: Are zeros from identifiable distinct processes?
  |
  Yes, and you can model them --> Zero-Inflated (ZIP/ZINB)
  Yes, but single clear process --> Hurdle model
  Unsure / prediction is the goal --> Hurdle model (more robust)

Step 3: Do you have enough data?
  |
  N < 100 --> Stick with NB regression
  100 <= N < 300 --> Hurdle model (simpler estimation)
  N >= 300 --> Compare ZINB and Hurdle; use diagnostics
```

### 7.2 Formal Diagnostic Tests

**Test for zero-inflation** (Van den Broek, 1995): A score test comparing the observed number of zeros to the number expected under a standard Poisson or NB model. If significant, there are more zeros than the base distribution predicts. This test only requires fitting the simpler model ([Van den Broek, 1995](https://pubmed.ncbi.nlm.nih.gov/7662855/)).

```python
def score_test_zero_inflation(y, mu_hat):
    """Van den Broek (1995) score test for zero inflation.

    H0: No zero inflation (standard Poisson is adequate)
    H1: Zero inflation present
    """
    n = len(y)
    observed_zeros = (y == 0).sum()
    expected_zeros = np.sum(np.exp(-mu_hat))

    numerator = (observed_zeros - expected_zeros) ** 2
    denominator = expected_zeros - n * np.mean(mu_hat ** 2 * np.exp(-mu_hat)) + expected_zeros**2 / n

    S = numerator / denominator
    p_value = 1 - scipy.stats.chi2.cdf(S, df=1)
    return S, p_value
```

**Vuong test** (comparing non-nested models): Commonly used to compare ZIP vs. Poisson or ZINB vs. NB. However, [Wilson (2015)](https://www.researchgate.net/publication/270913871_The_misuse_of_the_Vuong_test_for_non-nested_models_to_test_for_zero-inflation) demonstrated that the standard Vuong test is **misused** in this context because a non-zero-inflated model is nested (not non-nested) within its zero-inflated counterpart at the boundary. [Desmarais & Harden (2013)](https://journals.sagepub.com/doi/10.1177/1536867X1301300408) provide AIC- and BIC-corrected versions that reduce bias. Use the corrected version if you must run this test.

### 7.3 Information Criteria Comparison

Compare all candidate models using AIC and BIC:

```python
models = {
    'Poisson': poisson_model,
    'NegBin': nb_model,
    'ZIP': zip_model,
    'ZINB': zinb_model,
    # Hurdle requires combining log-likelihoods from both parts
}

for name, model in models.items():
    print(f"{name:12s}  AIC={model.aic:8.1f}  BIC={model.bic:8.1f}  LogLik={model.llf:8.1f}")
```

Lower AIC indicates better predictive performance (balancing fit and complexity). Lower BIC favors more parsimonious models. When AIC and BIC disagree, prefer BIC for small samples (it penalizes complexity more) and AIC when prediction is the primary goal ([Campbell, 2021](https://besjournals.onlinelibrary.wiley.com/doi/full/10.1111/2041-210X.13559)).

For the hurdle model, the log-likelihood is the sum of the binary component's log-likelihood and the truncated count component's log-likelihood. AIC = -2*logLik + 2*k where k is the total number of parameters across both parts.

### 7.5 Practical Recommendation for Our Problem

For the article highlight prediction problem:

1. **Start with NB regression** as the baseline. It may be sufficient.
2. **Test for zero-inflation** using the score test. If significant, proceed to step 3.
3. **Fit a Hurdle-NB model** as the primary zero-handling approach. The interpretive advantage is clear: the binary part models "did the user engage?" and the count part models "how much?"
4. **Fit ZINB for comparison** using `reading_progress` in the inflation equation.
5. **Compare via AIC/BIC and RQR diagnostics**. Choose the model with the best fit.
6. If models are comparable, **prefer the Hurdle** for its interpretability and robustness to zero-deflation.

## 8. Evaluation Metrics for Zero-Inflated Count Data

Standard regression metrics (R-squared, RMSE) are poorly suited to zero-inflated count data. Here are the appropriate metrics, organized by what they measure.

### 8.1 Overall Predictive Accuracy

**Root Mean Squared Error (RMSE)**: Penalizes large errors heavily. For zero-inflated data, RMSE is dominated by errors on the positive counts (where values are larger), which may cause the model to overpredict zeros to minimize RMSE.

**Mean Absolute Error (MAE)**: More robust to outliers than RMSE. For zero-inflated data, MAE gives equal weight to each observation regardless of magnitude. A model that always predicts 0 could have a deceptively low MAE if most observations are zero.

**Recommendation**: Report both, but weight MAE more heavily for zero-inflated data. Also report MAE separately for zeros and positive observations.

### 8.2 Likelihood-Based Metrics

**Log-likelihood**: The most principled evaluation for probabilistic count models. Higher (less negative) values indicate better fit. The full log-likelihood accounts for the entire predicted distribution, not just the point prediction.

**AIC/BIC**: Penalized log-likelihood metrics for model comparison (Section 7.3).

**McFadden's Pseudo-R-squared**: For count models, defined as 1 - (logLik_model / logLik_null). Values of 0.2-0.4 indicate excellent fit for count models -- much lower than the R-squared values expected in OLS ([McFadden, 1974](https://eml.berkeley.edu/reprints/mcfadden/zarembka.pdf)).

### 8.3 Probabilistic Scoring Rules

**Continuous Ranked Probability Score (CRPS)**: Generalizes MAE to probabilistic forecasts. For a predicted distribution F and observed value y:

```
CRPS(F, y) = integral_{-inf}^{inf} [F(x) - 1(x >= y)]^2 dx
```

For discrete distributions (counts), this becomes a sum. CRPS rewards sharp, well-calibrated predictive distributions ([Lokad CRPS reference](https://www.lokad.com/continuous-ranked-probability-score/)).

**Dawid-Sebastiani Score**: Based on the predictive mean and variance: DSS = (y - mu)^2 / sigma^2 + log(sigma^2). This explicitly penalizes miscalibrated variance.

### 8.4 Zero-Specific Metrics

These metrics focus on how well the model handles the zero-inflation component:

**Zero prediction accuracy**: What fraction of true zeros does the model correctly predict as zeros?

**Calibration of zero probability**: Compare predicted P(Y=0) with observed zero fraction across bins of the predicted probability.

```python
def zero_calibration(y, predicted_zero_prob, n_bins=5):
    """Check if predicted zero probabilities match observed zero rates."""
    bins = pd.qcut(predicted_zero_prob, q=n_bins, duplicates='drop')
    cal = pd.DataFrame({
        'predicted_p_zero': predicted_zero_prob,
        'actual_zero': (y == 0).astype(int),
        'bin': bins
    })
    summary = cal.groupby('bin', observed=True).agg(
        mean_predicted=('predicted_p_zero', 'mean'),
        mean_observed=('actual_zero', 'mean'),
        n=('actual_zero', 'count')
    )
    return summary
```

### 8.5 Concordance and Discrimination

**Concordance Index (C-statistic)**: For all pairs of observations where the outcome differs, what fraction have the higher predicted value for the higher observed count? C = 0.5 is random; C > 0.7 is acceptable. Related to Kendall's tau via C = (tau + 1) / 2.

**AUC for binary engagement**: Treat the problem as binary (any highlights vs. none) and compute ROC-AUC. This is the single most informative metric for a zero-inflated setting because the primary challenge is separating zeros from non-zeros.

### 8.6 Cross-Validation Strategy

For small datasets, use **stratified K-fold cross-validation** (K=5 or K=10) with stratification on the binary engagement indicator (has_highlights). This ensures each fold has a representative proportion of zeros and positives.

```python
from sklearn.model_selection import StratifiedKFold

y_binary = (y > 0).astype(int)
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

for train_idx, test_idx in cv.split(X, y_binary):
    X_train, X_test = X[train_idx], X[test_idx]
    y_train, y_test = y[train_idx], y[test_idx]
    # Fit and evaluate
```

For very small N (<100), use **leave-one-out cross-validation** (LOOCV) to maximize training data in each fold.

### 8.7 Summary of Recommended Metrics

| Metric | What It Measures | Priority |
|--------|-----------------|----------|
| AUC (binary) | Discrimination: separating zeros from positives | Primary |
| MAE (overall) | Average prediction error | Primary |
| MAE (positives only) | Count accuracy for engaged articles | Primary |
| Log-likelihood | Probabilistic fit quality | For model comparison |
| McFadden pseudo-R^2 | Explanatory power | For model comparison |
| CRPS | Full distributional calibration | Secondary |
| C-index | Pairwise ranking accuracy | Secondary |
| Zero calibration plot | Zero probability calibration | Diagnostic |

## 9. Practical Recommendations for Hundreds of Observations

### 9.1 What Works at This Scale

With ~200-500 articles, the statistical landscape constrains what is feasible:

**Tier 1 -- Always viable (N >= 50)**:
- Spearman/Kendall rank correlations between scores and highlight counts
- Negative Binomial GLM with 1-3 predictors
- Logistic regression for binary engagement (any highlights vs. none)
- Tier-level calibration (Low/Medium/High score bands vs. engagement rates)
- Bootstrap confidence intervals for all estimates

**Tier 2 -- Viable with care (N >= 150)**:
- Hurdle model (logistic + truncated NB)
- NB with up to 6-8 predictors (info_score, sub-dimensions, word_count, reading_progress)
- Per-dimension correlation analysis
- 5-fold cross-validated AUC

**Tier 3 -- Requires 300+ observations**:
- ZINB with separate inflation covariates
- Embedding features in count models (even after PCA reduction)
- Gradient-boosted two-stage model
- Formal Vuong tests and comprehensive model comparison

**Tier 4 -- Not recommended below 1000**:
- Deep learning approaches (neural network count models)
- VAE-based generative models
- End-to-end embedding-to-count models

### 9.2 Bayesian Estimation for Small Samples

When sample size is limited, [Neelon et al. (2010)](https://pmc.ncbi.nlm.nih.gov/articles/PMC3039917/) demonstrate that Bayesian estimation outperforms maximum likelihood for zero-inflated models in terms of both bias and precision. Bayesian methods:

- Incorporate prior information to stabilize estimates
- Provide credible intervals that have correct coverage even for small N
- Avoid convergence issues that plague MLE for ZINB with small samples

Implementation with PyMC:

```python
import pymc as pm

with pm.Model() as zinb_model:
    # Priors
    beta = pm.Normal('beta', mu=0, sigma=1, shape=n_features)
    alpha = pm.HalfNormal('alpha', sigma=1)  # dispersion
    psi = pm.Beta('psi', alpha=1, beta=1)     # zero-inflation prob

    # Linear predictor
    mu = pm.math.exp(X @ beta)

    # ZINB likelihood
    y_obs = pm.ZeroInflatedNegativeBinomial(
        'y', mu=mu, alpha=alpha, psi=psi,
        observed=y
    )

    # Sample
    trace = pm.sample(2000, tune=1000)
```

[PyMC ZeroInflatedNegativeBinomial documentation](https://www.pymc.io/projects/docs/en/stable/api/distributions/generated/pymc.ZeroInflatedNegativeBinomial.html) provides the full API reference.

### 9.3 Regularization and Feature Selection

With limited data, every parameter costs precious degrees of freedom:

1. **Start simple**: info_score alone as the predictor. Does it predict engagement at all?
2. **Add confounders one at a time**: word_count, reading_progress, category
3. **Test sub-dimensions**: Do the 4 sub-dimensions outperform the aggregate info_score?
4. **Use AIC to penalize complexity**: More parameters must earn their place via improved fit

For embedding features, **never** use raw 768-dimensional vectors with fewer than 1000 observations. Reduce to 5-10 PCA components and treat them as supplementary features alongside the structured predictors.

### 9.4 The Practical Minimal Pipeline

For a team with hundreds of articles and limited statistical bandwidth, here is the minimum viable analysis:

```python
# 1. Assess the data
zero_pct = (y == 0).mean()
mean_y = y.mean()
var_y = y.var()
print(f"Zero%: {zero_pct:.1%}, Mean: {mean_y:.2f}, Var: {var_y:.2f}, Var/Mean: {var_y/mean_y:.1f}")

# 2. Spearman correlation (primary metric)
rho, p = scipy.stats.spearmanr(info_score, y)
print(f"Spearman rho = {rho:.3f}, p = {p:.4f}")

# 3. AUC for binary engagement

auc = roc_auc_score((y > 0).astype(int), info_score)
print(f"AUC (any highlights) = {auc:.3f}")

# 4. Negative Binomial regression
import statsmodels.api as sm
X = sm.add_constant(info_score)
nb = sm.NegativeBinomial(y, X, loglike_method='nb2').fit(disp=False)
irr = np.exp(nb.params[1])
print(f"IRR per 1-point score increase: {irr:.4f}")
print(f"IRR per 10-point increase: {np.exp(nb.params[1] * 10):.3f}")

# 5. Hurdle model (if N >= 100)
from sklearn.linear_model import LogisticRegression
gate = LogisticRegression().fit(info_score.reshape(-1, 1), (y > 0).astype(int))
pos_mask = y > 0
if pos_mask.sum() >= 10:
    X_pos = sm.add_constant(info_score[pos_mask])
    nb_pos = sm.NegativeBinomial(y[pos_mask], X_pos, loglike_method='nb2').fit(disp=False)
```

### 9.5 When to Collect More Data

The most impactful investment is often more data rather than fancier models. Key thresholds:

- **Below 100 scored+archived articles**: Results are exploratory. Wide confidence intervals. Focus on basic correlations and binary engagement analysis.
- **100-200 articles**: Hurdle model becomes reliable. Sub-dimension analysis is feasible. Can start detecting moderate effect sizes (rho >= 0.25).
- **300+ articles**: Full model comparison pipeline is viable. ZINB with inflation covariates. Embedding features (after PCA). Formal model selection.
- **500+ articles**: Machine learning approaches become competitive. Cross-validated model comparison is statistically powered.
